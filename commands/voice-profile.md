# Voice Profile — Customize the voice-stylist agent

The `pt-doots:voice-stylist` agent reads two layers of rules:

1. **Bundle** at `agents/voice-stylist/profile.md` — universal good-prose rules that ship with the plugin.
2. **User overlay** in your local user memory — personal touches that override or extend the bundle.

This command walks you through setting up (or refreshing) your overlay. Run it anytime — after install, after you notice the agent missing a phrase you hate, whenever your voice changes.

---

## Step 0: Resolve user memory path

The overlay lives in user memory, scoped to the current project workspace. Resolve the path:

```bash
WORKSPACE_KEY=$(echo "$PWD" | sed 's|/|-|g')
USER_MEMORY="$HOME/.claude/projects/${WORKSPACE_KEY}/memory"
```

If the directory doesn't exist, create it:
```bash
mkdir -p "$USER_MEMORY"
```

If the user is running from outside a project workspace (unusual), fall back to `~/.claude/memory/` and warn them the overlay will apply globally.

---

## Step 1: Show the bundle

Read `agents/voice-stylist/profile.md` from the plugin install. Tell the user:

> Here's the universal voice profile that ships with pt-doots. Your overlay can extend or override any of it. I'll walk you through the personal layer now.

Display a tight summary (not the full file) — three or four bullet points covering: banned phrases included, prefix scheme, em-dash rule, plain-verb mapping.

Then ask:

> Want me to walk through customizing your overlay now, or just point you at the bundle to read on your own?

If "walk through" → continue to Step 2. If "point me at it" → print the bundle path and exit.

---

## Step 2: Banned phrases (personal layer)

> Are there phrases you DON'T use that drive you nuts when you see them in drafts? Examples from the bundle: `coalesce`, `bypass`, `leverage`. Tell me yours — I'll add them.

Collect a list. For each, ask:

> "{phrase}" — replace it with what? (Or just strip it.)

Build a mapping. If the user has no additions, skip ahead.

---

## Step 3: Audience tiers

> Who do you mostly write to? Pick the ones that apply:
> - Devs (laid-back, lowercase, abbreviations OK)
> - Internal non-tech folks (business-casual, clean sentences)
> - External partners or customers (business-polished, no slang)

For each tier the user names, ask:

> Any quirks for this tier? (e.g. "always capitalize sentence starts," "no all-caps greetings," "use specific emoji/sign-off")

If the user mostly writes to one audience, that becomes the default tier the orchestrator passes to voice-stylist.

---

## Step 4: Signature emojis or closing tags

> Any custom emojis, sign-offs, or hype phrases the agent should preserve when it sees them in drafts?

Examples to prompt with: `:doots-softly:`, `aw yeah`, `easy work`, `lets gooo`. The agent shouldn't strip these — they're part of the user's voice.

---

## Step 5: Prefix scheme (rare override)

The bundle uses Conventional Comments style: `must:` / `should:` / `nit:` / `opinion:` / `idea:` / `question:` / `praise:`. Ask:

> The bundled prefix scheme is must/should/nit/opinion/idea/question/praise. Want to use these, or your own set?

Most users will keep the bundle. If they want a custom scheme, capture the labels and what each means.

---

## Step 6: Write the overlay files

Write to `$USER_MEMORY/`:

- `voice_overlay.md` — main overlay with frontmatter, the answers from Steps 2-5 organized into sections (banned phrases, audience tiers, signature tags, prefix overrides)
- Optionally split per-tier files if the user gave detailed quirks: `voice_tier_laid_back.md`, `voice_tier_business_casual.md`, etc.

Each file starts with:

```yaml
---
name: {file-name}
description: Personal voice overlay for the voice-stylist agent. Generated by /voice-profile on {date}.
type: user
---
```

After writing, also append a one-line pointer to `$USER_MEMORY/MEMORY.md` so future Claude sessions find the overlay:

```markdown
- [voice_overlay.md](voice_overlay.md) — personal overlay for the pt-doots voice-stylist agent
```

If `MEMORY.md` doesn't exist, create it with a minimal header.

---

## Step 7: Confirm and exit

Show the user the file paths that were written:

> Done. Voice overlay saved to:
> - `{path}`
>
> The voice-stylist agent will pick this up automatically on the next `/prs` invocation. Run `/voice-profile` anytime to refresh.

If the user wants to test it, suggest:

> Want to test it? Type a sample sentence with one of your banned phrases and I'll run it through voice-stylist.

If they say yes, spawn `pt-doots:voice-stylist` with their sample and show the output.

---

## Notes

- This command is read/write only on user memory and the bundled profile — no source code, no agents/, no commands/.
- Do NOT modify the bundle (`agents/voice-stylist/profile.md`). The bundle is plugin-shipped; user customizations go in the overlay only.
- Existing overlay files in user memory are NOT overwritten without confirmation — if they exist, ask "merge into existing, or replace?"
